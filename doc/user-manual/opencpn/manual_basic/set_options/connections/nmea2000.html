NMEA 2000

For NMEA 2000 RX (Receive Transmission), OpenCPN 5.10 supports the four standard Actisense N2K formats ((ASCII Raw, Ascii N2K, Actisense Raw and Actisense N2K) The YachtDevices "RAW" format is exactly the same as "Actisense ASCII Raw", so that is supported as well. This is the only native format supported for YD.

  • Please review Quick Start Guide > NMEA 2000

  • OpenCPN only transmits onto the NMEA 2000 network for basic network house-keeping functions such as address claims and responding to product information requests (PGN’s 60928 and 126996).

  • OpenCPN does not echo or multiplex NMEA 2000 messages.

  • OpenCPN core does not convert NMEA 2000 messages to NMEA 0183 sentences (may be performed by plugins).

  • Plugins provide NMEA 2000 Autopilot control, as most autopilots use vendor-proprietary N2K messages.

  • OpenCPN does not provide support for NMEA 2000 over Wi-Fi or Ethernet. There is no recognized standard for transmitting NMEA 2000 binary data over TCP/IP.

  • Many gateway devices connect directly to a NMEA 2000 network, convert the data from NMEA 2000 to NMEA 0183 and re-transmit over Wi-Fi or Ethernet using TCP or UDP. A benefit of this approach is that many other devices such as PCs & Tablets, running a variety of software such as OpenCPN or Navionics and connected to the gateway WiFi access point, can consume the converted NMEA 0183 data. See Supplementary Hardware Nmea Gateways and Nmea & Seatalk

For direct connection to NMEA 2000 networks

OpenCPN 5.8.x (forward) supports the following adapters on all platforms (Windows, Mac, Linux)

  • Actisense NGT-1, NMEA 2000 to USB Interface, Bidirectional.

  • Yacht Devices NMEA 2000 USB Gateway YDNU-02.

Additionally on Linux, any CAN Bus adapter that implements the SocketCAN interface is also supported. Some of these include:

  • Kvaser Leaflight

  • Canable Cantact 1

  • Rusoku Toucan Marine 2

  • Waveshare CAN Hat for Raspberry PI

  • USBTin

  • Peak Systems

  • Copperhill Tech PICAN-M for Raspberry Pi

1 If using standard firmware, acts as a Serial Line CAN device, (slcan), if using Candlelight firmware, acts as a native CAN device (can).

2 May require kernel modules to be built

On Linux, follow the manufacturer’s instructions, if required, to enable the device. For example on a Raspberry Pi, devices that use the MCP2515 chipset such as those from Waveshare & Copperhill, require entries similar to the following to be enabled in /boot/config.txt

dtparam=spi=on
dtoverlay=mcp2515-can0,oscillator= 12000000,interrupt=25,spimaxfrequency=1000000

Bringing up a native CAN device (eg can0) on Linux

Run the following command:

 sudo ip link set can0 type can bitrate 250000
 sudo ip link set up can0

Bringing up Serial Line CAN interface (eg. slcan0) on Linux

First of all, identify the interface:

 ls -la /dev/ttyA*

Note device name, may or may not be ttyACM0.

Bring up the daemon

 sudo slcand -o -s5 -t hw -S 1000000 /dev/ttyACM0
 sudo ip link set up slcan0

Automatically starting the SocketCAN service on Ubuntu.

Note this example is for a Serial Line CAN Device (slcan). In the /etc/systemd/system directory add the file socketcan.service containing the following:

 [Unit]
     Description=Enabling SocketCAN
 [Service]
     Type=oneshot
     RemainAfterExit=yes
     ExecStart=/usr/bin/slcand -o -s5 -t hw -S 1000000 /dev/ttyACM0
     ExecStart=/sbin/ifconfig slcan0 up
     ExecStart=/sbin/ifconfig slcan0 txqueuelen 1000
     ExecStop=/sbin/ifconfig slcan0 down
 [Install]
     WantedBy=default.target

lAutomatically starting the SocketCAN Service on a Raspberry Pi

For Debian Buster release, add the following entries to etc/network/interfaces Note the following example is for a native CAN device.

 #can interfaces
 auto can0
 iface can0 inet manual
 pre-up /sbin/ip link set can0 type can bitrate 250000 restart-ms 100
 pre-up /sbin/ifconfig can0 txqueuelen 1000
 up /sbin/ifconfig can0 up
 down /sbin/ifconfig can0 down

For Debian Bullseye release, add the following entries to etc/network/interfaces Again, note the following example is for a native CAN device.

Check if there is a file /etc/network/interfaces with the following content:

# Include files from /etc/network/interfaces.d:
source-directory /etc/network/interfaces.d

Create a file /etc/network/interface.d/can0 with the following content:

#physical can interfaces
auto can0
iface can0 can static
bitrate 250000
pre-up /sbin/ip link set can0 type can restart-ms 100
down /sbin/ip link set $IFACE down
up /sbin/ifconfig $IFACE txqueuelen 10000

Bringing up a Virtual CAN interface (eg. vcan x ) on Linux

Run the following commands in a terminal:

 modprobe vcan
 sudo ip link add dev vcan0 type vcan
 sudo ip link set up vcan0

  • The Virtual CAN device is useful for testing OpenCPN without a NMEA 2000 network and for replaying candump files.

  • On Linux the canutils are a useful set of tools for logging and replaying data from the NMEA 2000 Network.

Installing canutils

Run the following command:

 sudo apt-get install can-utils

Logging data from a native can device

Run the following command:

 candump can0 -l

Replay a log file

To replay a log file named candump-2023-04-13_141102.log with data recorded from can0 and to replay infinitely on the virtual can interface vcan0, use the following command:

 canplayer -I candump-2023-04-13_141102.log vcan0=can0 -l i -v

  • 126996, NMEA Product Information

  • 127245, Rudder 1

  • 127250, Heading

  • 127257, Attitude (Roll, Pitch) 1

  • 127488, Engine Parameters, Rapid Update2

  • 127489, Engine Parameters, Dynamic2

  • 127505, Fluid Levels2

  • 127508, Battery Status2

  • 128259, Speed 1

  • 128267, Depth 1

  • 128275, Distance Log 1

  • 129025, Position (Rapid Update)

  • 129026, COG/SOG (Rapid Update)

  • 129029, GNSS Position

  • 129540, GNSS Satellite Data 1

  • 129038, AIS Class A Position Report

  • 129040, AIS Class B Position Report

  • 129041, AIS AToN

  • 129793, AIS Date & Time Report

  • 129794, AIS Class A Static Data & Voyage Report

  • 129798, AIS SAR Position Report

  • 129809, AIS Class B Static Data Report, Part A

  • 129810, AIS Class B Static Data Report, Part B

  • 130306, Wind 1

  • 130310, Environment (Sea & Air Temperature, Barometric Pressure) 1

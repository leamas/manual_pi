Use GPX Files

GPS Exchange Format (GPX) is a file format that enables devices to transfer GPS data (marks, waypoints, routes and tracks) with one another. GPX files are text files formatted using Extensible Markup Language (XML). Modern commercial chartplotters such as Garmin, Raymarine, Navico (B&G, Simrad, Lowrance), Furuno and many applications such as Navionics, Coastal Explorer, PredictWind, and LuckGrib support GPX format. This allows route creation and editing in offline tools (manual or automatic weather routing), followed by a transfer to the boat network for use during navigation. It also allows sharing of routing or track information between users.

Note: OpenCPN provides a feature for backing up navigation data. See Navigation Data Backup. The most current backup file is named "navobj.xml", located in the OpenCPN conf/ini folder. NavObj files may be imported in the same manner as any gpx file, by selecting the Import GPX button in the Route & Mark Manager.

Tracks, Routes, Waypoints & Marks in OpenCPN

A track is a record of the vessel’s position as it proceeds. A trail is left behind the vessel on the chart display. The record includes distance, bearing, latitude, longitude, speed and a timestamp for each trackpoint recorded. OpenCPN can create them automatically for you when you are sailing.

A route is a predefined path or passage from one point to another. Routes generally use a series of Marks or Waypoints at each course change.

From tracks, you may create and save routes for making the same trip again sometime. Routes may also be directly created in the map canvas with OpenCPN’s Create Routes .
You may also create or manipulate routes with external (planning) tools.

Marks are user defined markers showing points of interest, which can serve many purposes. They could mark good anchorages, fishing spots, new navigational markers (wrecks, channel markers) or anything else of interest,

Waypoints are an element in gpx files that contain all the information necessary to define a point on a map, along with supporting information (symbol, name, description, and optional extension information).

OpenCPN saves tracks, routes and mark in the navobj.xml using the following GPX elements: <rte> and <rtept> for routes and waypoints, <trk> and <trkpt> for tracks and points in a track, and <wpt> for marks.

Routes, tracks, waypoints and marks are imported, exported and saved as gpx (GPS Exchange Format) files, a specific XML format. OpenCPN uses the Garmin GPS Extensions V3 schema.

See Terminology > Marks and Waypoints Technical: Routes, Marks, GPX and GUID.

GPX File Structure

Here is a generalized gpx file structure. It is important to understand that the Body Content can contain many elements, such as a series of waypoints, or all the navigation markers in an area, etc.

<?xml version="1.0"?>
- Header
- Body  (e.g. a route)
  - Body Extensions (for additional route properties)
  - Body Content (e.g. route waypoints, aka route points)
    - Body Content Extensions (for additional waypoint properties)

Header (mandatory)

The header contains information about the gpx file’s structure for the program. The header information section is mandatory. When developing your own layers, you can copy an OpenCPN gpx file header with the necessary information and “reuse” it for other layers you may produce. Simply use Route & Mark Manager→Export selected and have a look at it in a text editor.

<gpx version="1.1" creator="OpenCPN" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
xmlns="http://www.topografix.com/GPX/1/1" xmlns:gpxx="http://www.garmin.com/xmlschemas/GpxExtensions/v3"
xsi:schemaLocation="http://www.topografix.com/GPX/1/1 http://www.topografix.com/GPX/1/1/gpx.xsd
http://www.garmin.com/xmlschemas/GpxExtensions/v3 http://www8.garmin.com/xmlschemas/GpxExtensionsv3.xsd"
xmlns:opencpn="http://www.opencpn.org">
//Body goes here//
</gpx>

Body - Route

In this Route file, the body contains the <rte> element, with its own extensions, then the content of the route which comprises a series of route waypoints (<rtept>). Splitting or joining routes is relatively easy to do, with the XML text.

<rte>
    <name>Test Route</name>
    <desc>My test route</desc>
    //Route Extensions goes here//
    <rtept lat="26.261414676" lon="-77.388078481">
    //Route Point Extensions go here//
    </rtept>
    //additional Route Points, as necessary, go here//
</rte>

Body - Route Extensions

The extensions to the Route file are specific to OpenCPN and are not part of the Garmin schema.

Note: GUID (Globally Unique Identfier) - it should not be copied and reused as it uniquely identifies an object (in this case this route). GUIDs are created by the program from a combination of current time and current internet location. It is entirely possible to create your own by incrementing (in hexadecimal) an existing GUID.

Note: OpenCPN will ignore GUID during import of a gpx file. Instead, it uses the Name and the position values (lat/lon) to determine uniqueness.

<extensions>
    <opencpn:guid>29b00000-0298-4378-bb6b-2f3c7aa4ffff</opencpn:guid>
    <opencpn:viz>1</opencpn:viz>
    <opencpn:start>Key Biscayne</opencpn:start>
    <opencpn:end>Miami</opencpn:end>
    <opencpn:planned_speed>6.00</opencpn:planned_speed>
    <opencpn:planned_departure>2024-05-27T21:59:11Z</opencpn:planned_departure>
    <opencpn:time_display>PC</opencpn:time_display>
    <gpxx:RouteExtension>
        <gpxx:IsAutoNamed>false</gpxx:IsAutoNamed>
    </gpxx:RouteExtension>
</extensions>

Body Content - Route Point

The Route Point element is a waypoint with position, name, symbol properties. Route Points are a specific instance of Waypoint type.

<rtept lat="26.261414676" lon="-77.388078481">
    <time>2024-05-27T21:51:52Z</time>
    <name>001</name>
    <sym>diamond</sym>
    <type>WPT</type>
    //Route Point extensions go here//
</rtept>

Body Content Extensions - Route Point Extensions

Each Route waypoint in OpenCPN gpx files has additional properties that are stored in the Extensions. These properties are editable in the right-click menu Waypoint Properties>Extended as long as the waypoint is not in a Layer.

<extensions>
    <opencpn:guid>434dffff-57b0-45e6-85dd-71e62ca70000</opencpn:guid>
    <opencpn:auto_name>1</opencpn:auto_name>
    <opencpn:arrival_radius>0.050</opencpn:arrival_radius>
    <opencpn:waypoint_range_rings visible="false" number="0" step="1" units="0" colour="#FF0000" />
    <opencpn:scale_min_max UseScale="false" ScaleMin="2147483646" ScaleMax="0" />
</extensions>

Marks (or Points Of Interest)

Overview Use Marks.

Marks are also used to represent sailing-related information like position of wrecks, buoys, bridges, locks and anchoring or mooring spots and could also contain information about less "sailing related" information such as a fueling station or a diving spot. An exported Mark will appear in the gpx file as element <wpt>:

<?xml version="1.0"?>
<gpx version="1.1" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xmlns="http://www.topografix.com/GPX/1/1">
    <wpt lat="53.162153333" lon="5.931996667">
        <time>2024-06-02T11:25:51Z</time>
        <name>Pier Pander
        (tel: 31-1234567890)</name>
        <desc>Fuel Station Pier Pander
        Different kinds of fuel and water.</desc>
        <sym>Service-Fuel-Pump-Diesel</sym>
        <type>WPT</type>
    </wpt>
</gpx>

Each mark has supporting "connected" data, including coordinates, symbol, name, and text description.

The coordinates of the mark (latitude and longitude) are expressed in decimal degrees using the WGS 84 datum.

lat="53.162153333" lon="5.931996667"

Every mark has a symbol connected to it, to be placed on the map at the coordinates.

<sym>Service-Fuel-Pump-Diesel</sym>

Besides the default symbol set (see Mark Properties→Icon), user-defined symbols may be created and stored in the UserIcons folder located under the folder where OpenCPN keeps its conf/ini file. See Advanced User Manual - Icons Library - Use your own Icons for more information.
If OpenCPN cannot find an icon file with the name you put in the gpx file, OpenCPN will substitute a circle with a dot in the middle.
If you distribute your gpx file containing custom symbol names to other users, please distribute the associated icon files WITH the gpx file.
When naming custom symbols, please think of a "unique" name in doing so, as more producers may have the same idea. For example, layers produced by the Openwaterkaart-project use symbols with names starting with ow_ so please use another unique combination for your own to prevent your symbols being overwritten by (or overwrite) others when using layers of different suppliers.

OpenCPN displays a name beside the Symbol. As an example, marks of fueling stations, you could have the name of the station and/or the phone number in that text. For the users of that gpx-file, the necessary information would be visible directly on the map. The text displayed can be multiline, simply by inserting new line character in the gpx file at the appropriate spot with an Enter key.

In OpenCPN Layers, you can select if the mark name is visible or not by checking the Show WPT Names check box in Route & Mark Manager→Layers.

Beside the name displayed on the map, you can also create a description for a mark. This text is NOT displayed on the map, but users can view the properties of the mark to see it. In the example of the fueling station, you could add what is available at the fueling station, e.g. water, diesel, gas, etc.

Optional Mark Extensions

There is a way to attach some extra functionality to the marks, either through Route & Mark Manager→Marks_Properties→Extended, or by directly editing the gpx file. These are OpenCPN-specific and are added to the standard mark properties in the form of extensions. Again, the GUID must be unique to each mark, unless the mark is being imported.

<extensions>
    <opencpn:guid>3b480000-2b02-4517-b442-48dc5e35ffff</opencpn:guid>
    <opencpn:viz_name>1</opencpn:viz_name>
    <opencpn:auto_name>1</opencpn:auto_name>
    <opencpn:arrival_radius>0.050</opencpn:arrival_radius>
    <opencpn:waypoint_range_rings visible="true" number="2" step="5" units="1" colour="#FF0000" />
    <opencpn:scale_min_max UseScale="false" ScaleMin="2147483646" ScaleMax="0" />
    <opencpn:tidestation>Pelican Harbour, Abaco Island, Bahamas</opencpn:tidestation>
</extensions>
Mark Range Rings

With this extension you can put colored circles around the specific waypoint to make it more visible compared to other waypoints.

With the visible="true" part you can switch "on" this extension.
With number="2" you create two circles around the waypoint with the color identified in colour="#FFFFFF".
With units="0", the ring size units are nautical miles, or ="1" for kilometers. With step="1", the there is one of the units between the rings, e.g. 1 km.

Hexadecimal color values may be evaluated at Color Converter.

Dynamic Scaling

Waypoints can be "dynamically" shown or hidden with this extension. Use UseScale="true" you can switch "on" this extension.
This example shows the waypoint up to scale 1:100,000 (shown in the lower right corner of OpenCPN).
Above a scale of 1:100,000, set by ScaleMin="100000", the waypoint is suppressed and thus not visible on the map.
Apparently only ScaleMin in this extension is used by OpenCPN as the "upper" scale-limit for visibility of the waypoint.

Tide Stations

You may add a local tide station to any waypoint. Users can then check to tide status in right-click→Waypoint→Properties→Extended→Tide Station by selecting the icon.

Optional Route Extensions

Route and Track Colors

For route and track colors, OpenCPN follows this schema:

<rte>
  <extensions>
    <gpxx:RouteExtension>
      <gpxx:DisplayColor>Cyan</gpxx:DisplayColor>
    </gpxx:RouteExtension>
  </extensions>
</rte>

Note that hexadecimal colors are not allowed in this extension.

GPX Files as Layers

Gpx files that are not intended to be changed further may be placed in the Layers folder, a sub-folder of the folder containing the config/ini information of OpenCPN.
When placed in this specific folder, the layer is persistent (does not have to be imported in OpenCPN each startup). The name of the layer is determined from the filename of the gpx-file.
NOTE: new files in the Layers folder will only be used when OpenCPN is restarted.

So if you were to save the example above to a layer and call it "Fuel Stations Nederland.gpx" it would present itself in OpenCPN as a layer with the name Fuel Stations Nederland.

See how a gpx file can be made into a useful Layer in Advanced User Manual - Use Layers.

Custom Symbols

The waypoint symbols are the normal <sym> sub-element of the <wpt> element as defined by the GPX standard and OpenCPN by default supports the symbols from MarkIcons, plus any user-defined symbols you have in the Usericons folder.

In the GPX file, use the symbol’s filename without ".svg" as the value for <sym>.

See usericon for additional pre-made symbols.
See icons for more information on OpenCPN icons

GPX Editors

The key features of a good GPX editor include:

  • Opening and importing tracks from GPX files and viewing them on a digital map.

  • Switching views between digital maps and a geo-referenced satellite image.

  • Drawing tracks on the map using a mouse, or stylus pen, or even a finger (latter two require a touchscreen platform).

  • Adding, editing, moving, and removing waypoints

  • Reversing track direction and re-sequencing points.

  • Search capabilities to look up place names and points of interest.

  • Splitting a track into two parts to remove unnecessary sections.

  • Merging tracks to turn several recorded tracks into one unified track or a multi-track file (an advanced GPX feature).

  • Exporting the track to a GPX file for use in a navigation device or app.

  • Point-density reduction for conversion of tracks to routes.

  • Self-contained (not requiring any internet connection, unless for free maps & satellite images).

Very few tools have all these capabilities. The following sections provide some suggestions that may get you started.
